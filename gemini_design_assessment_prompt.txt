DESIGN SYSTEM ASSESSMENT REQUEST
==================================
Project: Intensely Mobile App
Design System: Version 1.0 (Created by Claude Code)
Assessment Date: 2026-01-13

CONTEXT:
--------
This is a React Native/Expo mobile fitness app called "Intensely" focused on High-Intensity Circuit Training (HICT). The design system was created by Claude Code and has been implemented across 10/24 screens.

YOUR TASK:
----------
Perform a comprehensive design system assessment covering:
1. Visual Design Quality

3. Color Contrast Ratios

ASSESSMENT REQUIREMENTS:
========================

1. COLOR CONTRAST ANALYSIS:
   - Calculate contrast ratios for ALL text/background combinations
   - Test against WCAG 2.1 Level AA (4.5:1 normal text, 3:1 large text)
   - Test against WCAG 2.1 Level AAA (7:1 normal text, 4.5:1 large text)
   - Flag any failing combinations
   - Test both light and dark modes

2. SPECIFIC COMBINATIONS TO CHECK:
   Light Mode:
   - Primary text (#0F172A) on white background (#FFFFFF)
   - Secondary text (#475569) on white background
   - Tertiary text (#94A3B8) on white background
   - Primary button text (white) on red background (#FF0000)
   - Success text on success background
   - Error text on error background

   Dark Mode:
   - Primary text (#F8FAFC) on dark background (#0F172A)
   - Secondary text (#CBD5E1) on dark background
   - Card text on elevated background (#1E293B)

3. TYPOGRAPHY ASSESSMENT:
   - Evaluate font size hierarchy
   - Check line height ratios
   - Assess readability at different sizes
   - Mobile optimization (minimum 16px body?)
   - Letter spacing appropriateness

4. SPACING & LAYOUT:
   - Consistency of 4px base scale
   - Touch target compliance (44px minimum)
   - Padding/margin patterns
   - Card spacing and hierarchy

5. VISUAL DESIGN QUALITY:
   - Color palette harmony
   - Brand appropriateness for fitness app
   - Energy level conveyed by colors
   - Outdoor readability considerations
   - Dark mode color choices

6. MOBILE UX PATTERNS:
   - Button sizing for thumb zones
   - Form input accessibility
   - Card design patterns
   - Navigation clarity
   - Loading states

7. COMPONENT CONSISTENCY:
   - Button variants (primary, secondary, ghost)
   - Card patterns (elevated, flat)
   - Text component usage
   - Input field patterns

8. IMPROVEMENT RECOMMENDATIONS:
   - List specific, actionable improvements
   - Prioritize by impact (High/Medium/Low)
   - Provide alternative color values if needed
   - Suggest spacing adjustments
   - Typography refinements
